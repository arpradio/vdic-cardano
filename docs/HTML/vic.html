1. Overview

1.1 Purpose

The VDIC specification defines a standard protocol for integrating verifiable, decentralized off-chain storage with Cardano dApps. It provides configurable options for different use cases while maintaining data integrity and verifiability.

1.2 Key Features

  • ✓ Verifiable Data Integrity: Cryptographic proofs ensure data hasn't been tampered with
  • ✓ Decentralized Storage: Multiple independent nodes ensure data permanency
  • ✓ Flexible Configuration: Adapt to different use cases (public, private, encrypted)
  • ✓ Cardano Integration: Native support for DIDs, VCs, and asset-based access control
  • ✓ Trustless Access: Light clients can verify data without trusting intermediaries

5.1 SDK Usage Example

TypeScript Integration
import { VDICClient, CardanoProvider } from '@vdic/cardano-sdk'; // Initialize VDIC client const vdic = new VDICClient({ configPath: './vdic-config.json', cardanoProvider: new CardanoProvider({ network: 'mainnet' }) }); // Store content async function storeContent(file: File, metadata: any) { const result = await vdic.store({ content: file, metadata, accessControl: { type: 'token_gated', requiredAssets: ['asset1abc...'] } }); return result.cid; } // Retrieve content async function getContent(cid: string, userWallet: any) { const hasAccess = await vdic.verifyAccess(cid, userWallet); if (hasAccess) { return await vdic.retrieve(cid); } throw new Error('Access denied'); }

7. Security Considerations

7.1 Data Protection

  • ⚠ Encryption at Rest: All sensitive content should be encrypted before storage
  • ⚠ Key Management: Use hierarchical deterministic keys linked to Cardano assets
  • ⚠ Access Control: Implement zero-knowledge proofs for privacy-preserving access

7.2 Network Security

  • ⚠ Node Verification: All nodes must have verified DIDs and valid VCs
  • ⚠ Communication Security: Use TLS for all API communications
  • ⚠ Consensus Mechanisms: Implement byzantine fault tolerance for critical operations

7.3 Compliance

  • ⚠ GDPR Compliance: Implement data deletion mechanisms for personal data
  • ⚠ Audit Trails: Maintain verifiable logs of all data operations
  • ⚠ Data Sovereignty: Respect geographic restrictions on data storage
